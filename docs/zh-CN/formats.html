格式

lhasa 读什么 — 三家族十格式。

.lzh — LHA / LZH / LHarc（1988-）

LHA（前身 LHarc）是 1988 年 Yoshizaki 起的格式，用 LZARI / LZHUF 算法（LZ77 + Huffman）。压缩方式写在文件后缀里：-lh5- 之类。lhasa 读全部 LZH 方法：

后缀算法说明
-lh0-不压缩 归档头里仍记 method。
-lh1-LZ77 + 算术编码 (LZARI) 原 LHarc 方法，少见。
-lh4-LZ77 + 静态 Huffman LZARI 变体。
-lh5-LZHUF (LZ77 + 动态 Huffman) 默认。现在遇到的 .lzh 多数是它。
-lh6-静态 Huffman 变体 压缩率更高，工具支持窄一点。
-lh7-LZHUF + tree 变体 实验性。
-lhx-LZH 兼容性标记 用更新约定 + 分开的 -lhd- 目录头。

除了压缩方法，lhasa 还处理 LZH 头变体：level-0/1/2（文件名长度、CRC、 扩展时间戳）、-lhd- 目录扩展头、MacBinary 编码成员。

.lzs — LArc（1988-）

LArc 是 Okumura / Yoshizaki 的另一个归档工具，跟 LHarc 同时代。 方法标记是 -lz5- / -lzs-。底层还是 LZARI， 但格式结构跟 LHA 完全不同（魔数不同、文件名后缀不写方法）。

lhasa 同时读老 LArc 和现代 -lz5-（LZ5）方法。

.pma — PMarc（1989-）

PMarc 是 1980 年代末日本 NEC PC-98 平台流行的 LHA 家族工具。两种方法 （-pm1- / -pm2-）配稍不同的头布局，底层 也是 LZARI。PC-98 软件分发的 .pma 文件是典型用例。

lhasa 不读什么

  • zip / gzip / bzip2 / xz / zstd / 7z / rar——用对应工具。 跟 lhasa 的格式集没有重叠。
  • 以上任何格式的 encoder——lhasa 严格只做 decoder。
