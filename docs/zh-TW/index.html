lhasa

純 FOSS 的 LHA / LZH / PMarc 解碼器，單檔自包含靜態二進位發佈。

lhasa 內部 vendored 自 fragglet/lhasa ——ISC 授權的函式庫與 CLI，作者 Simon Howard。我們不維護解碼器； 我們只把它打包成跨平台零相依的靜態二進位。

macOS / Linux / Windows 安裝 看四個常用指令

~200 KB 靜態二進位 · 任何 Linux / macOS / Windows 都不需 libc / zlib / libiconv。

最新 release · upstream · 資安稽核 · 支援的格式 · 用法 · 安裝 · 建置稽核

一句話。lhasa 是 lha 命令列工具的純 FOSS 復刻——讀 .lzh（LHA）、.lzs（LArc）、 .pma（PMarc）歸檔，沒有歷史上 lha 的非自由 再分發包袱。上游是 fragglet/lhasa （ISC，Simon Howard）；本倉 （ljh-sh/lhasa） 是多架構靜態二進位分發。CLI 二進位名稱故意是 lha（不是 lhasa），所以老指令碼 lha x foo.lzh 直接能跑。 誰該用：在沒裝 1990 年代歸檔工具的系統上解 30 年前 .lzh 的人；CI 流水線要一個 200 KB、零外部相依（libz / libiconv / libcurl 都不要）的二進位；要用 liblhasa 在 C 程式裡嵌入解碼的人。本站是什麼： 安裝指南（x eget / apt / brew / 源碼編譯）、 用法導覽（四個常用指令 l / x / t / p）、 格式目錄（.lzh / .lzs / .pma 三家族）、源碼級資安稽核 （關於 vendored 解碼器）、建置稽核（CI 矩陣、SHA256SUMS、可重現性）。本站不是什麼：不是 encoder、不是 lzh 格式擴充、不是從零寫的 lhasa-rs。
