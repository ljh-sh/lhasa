インストール

3 つのインストール方法——最初は静的バイナリ、次にディストロのパッケージマネージャ、最後にソースからビルド。

1. 静的バイナリ（1 行）

最も速く最も移植性のある方法。x eget を使う—— x-cmd の GitHub release パイプラインのラッパー：

x eget ljh-sh/lhasa    # ~200 KB、ゼロ依存、マルチアーキ静的ビルド

インストールされる CLI の名前は lha（lhasa ではない—— プロジェクトはわざと元の LHa for UNIX コマンドをミラーしています）。インストール後：

$ lha --help | head -1
Lhasa v0.6.0 command line LHA tool  - Copyright (C) 2011-2025 Simon Howard

2. ディストロのパッケージマネージャ

x install lhasa を使うと、x-cmd がディストロ公式の lhasa パッケージに解決します——ljh-sh/lhasa と同じソース：

x install lhasa    # apt / brew / pacman / zypper / xbps-install / pkg / pkgin

3. ソースからビルド

ソースは upstream/lhasa/ にあり、 git subtree で fragglet/lhasa から取得：

git clone https://github.com/ljh-sh/lhasa.git
cd lhasa
bash scripts/build.sh
bash scripts/smoke.sh
